# CODEX.md

## Project Goal

《临渊行》当前以“未来设计师·全国高校数字艺术设计大赛 1-C1 交互网页设计”为参赛方向，目标为国赛二等奖。

本项目比赛提交质量优先于长期工程化。2026年6月20日前，不进入后端开发，不做真实业务系统。

## Current Project State

当前项目是纯静态 HTML / CSS / JavaScript 概念游戏官网。

已完成：

- 首页 `index.html`
- 角色档案页 `characters.html`
- 4 个角色详情页
- 剧情档案页 `archive.html`
- 2 个剧情详情页
- 世界观档案库 `archives.html`
- 参赛演示模式 `demo.html`
- 新闻、预约、登录、404 静态页面
- PV 播放、角色弹窗、地图灯箱、Lore 折叠、世界观档案静态筛选
- README、阶段日志、C1 信息架构、提交检查清单、资源说明

## Codex Working Rules

Codex 后续工作必须遵守：

- 先读 README、`docs/stage-log.md`、`docs/c1-submission-checklist.md`
- 改代码前先说明改动范围
- 不主动扩大功能范围
- 不接入真实业务功能
- 不大改首页视觉气质
- 不为了凑页面数量制造空页面
- 不删除已有用户改动
- 修改后必须做链接、响应式、资源路径和静态边界检查

## Strict Boundaries

比赛提交前禁止：

- 后端
- 数据库
- API 请求
- 真实登录
- 真实预约
- 用户数据收集
- CMS
- 后台管理
- 评论、收藏、搜索、筛选、分页
- 真实下载、支付、短信、邮件、上传
- 大规模框架迁移
- 大规模视觉重做

参赛演示模式 → 首页首屏 → PV → 地图寻迹 → 世界观档案库 → 角色档案 → 剧情档案 → Project 说明

## Final Check

每次改动后至少检查：

- 页面是否仍可静态打开
- 链接是否可达
- 资源是否为相对路径
- 移动端是否可读
- 是否破坏现有交互
- 是否出现真实业务能力暗示
- 是否与国赛二等奖目标一致
- 是否满足 3 分钟宣讲脚本、匿名投稿和人工整理参赛材料清单要求
