# Project Case Study

## Project Overview

`临渊行` 是一个以国风志怪叙事为核心气质的静态概念游戏官网案例。项目目标不是模拟完整游戏业务，而是通过一套多页面静态官网结构，展示世界观包装、内容分层、页面节奏控制与前端落地能力。

## Background

这个项目以“作品集项目”而不是“商业官网项目”为前提启动。背景需求并非上线真实产品，而是围绕一个虚构的叙事游戏题材，做出一套更接近正式游戏官网观感的静态展示页，用来呈现内容组织能力、视觉统一性和静态交互表达能力。

## Design Goal

设计目标主要有三点：

- 让首页拥有明确的主视觉吸引力与世界观辨识度
- 让访客能够快速理解角色、场景、设定与玩法概念之间的关系
- 在不引入真实业务系统的前提下，让项目仍具备完整、专业的概念官网表达

## Information Architecture

项目采用多页面静态结构，以首页为核心，子页面补充边界明确的说明型内容。

- `index.html`
  承担主叙事展示，包括 Hero、PV、角色、地图寻迹、剧情档案、Lore、玩法概念与 Project 区。
- `news.html`
  用于展示阶段更新、公告与制作方向说明，建立“官网持续更新”的内容层次。
- `reserve.html`
  以静态方式模拟未来预约入口的视觉结构，但不提供真实表单提交。
- `login.html`
  以静态方式展示未来账号入口方向，但不提供任何真实认证逻辑。
- `404.html`
  用于补齐站点完整性与错误访问场景，强化整体网站闭环感。

## Visual Direction

视觉方向围绕“克制、悬疑、旧事未尽”的国风志怪氛围展开。页面并不追求复杂动画或商业化强转化设计，而是强调：

- 主视觉情绪统一
- 世界观文本与画面协同
- 角色与场景的叙事感表达
- 让 Project 区能够自然承担作品集说明职责

整体风格更偏向概念包装与叙事展示，而非功能型门户网站。

## Interaction Design

交互设计保持在“静态展示增强”的范围内，避免超出项目边界。

- PV 区提供基础播放触发
- 角色区提供片段展开与人物资料弹层
- 地图寻迹区提供灯箱式场景预览
- Lore 区通过折叠结构增强阅读层次
- 多页面入口保证浏览路径完整

这些交互主要服务于叙事展示和信息编排，而不是服务于真实业务流程。

## Technical Implementation

技术实现严格保持纯静态前端：

- HTML 负责页面结构与内容分层
- CSS 负责整体视觉、响应式布局与展示细节
- JavaScript 负责基础交互，如视频触发、弹层、折叠与轻量状态切换
- GitHub 负责版本管理与项目展示
- Netlify 用于静态站点演示部署

项目没有迁移到 React、Vue、Next.js、Vite 或其他框架，也没有接入后端服务。

## Development Boundary

为了保持项目真实、可信且适合作品集展示，开发边界被明确限制在静态概念官网范围内。

- 不接入真实登录
- 不接入真实预约
- 不接入数据库
- 不接入 CMS
- 不接入后台管理
- 不接入 API
- 不收集用户信息
- 不声称为真实上线游戏或真实运营官网

这意味着项目更接近“概念表达与前端展示案例”，而不是“具备业务闭环的产品系统”。

## Problems Solved

这个项目主要解决了几个作品集层面的实际问题：

- 让一个纯视觉概念，变成可以被浏览和理解的完整网站结构
- 把世界观、角色、玩法概念与项目说明组织成清晰的信息架构
- 在不依赖框架和后端的前提下，完成较完整的多页面静态体验
- 让 Project 区承担“自我说明”作用，帮助项目更适合放入简历、作品集与 GitHub
- 通过 404、新闻、预约、登录等静态子页面补齐完整感，而不制造真实业务假象

## What I Learned

这个项目带来的核心收获不只是页面实现本身，更包括作品集表达层面的经验：

- 叙事型官网需要先理清信息架构，再决定视觉堆叠顺序
- 静态项目也需要明确边界，否则很容易让展示与真实能力不匹配
- Project 区和 README 同样属于作品的一部分，而不只是附属说明
- 即使不接入真实系统，也可以通过内容组织和交互细节提升项目完成度
- 对于概念项目，真实表达比夸大更重要

## Future Plan

后续计划仍会围绕“展示质量提升”而不是“功能扩张”推进。

- 补充更完整的项目截图素材
- 继续扩展新闻、世界观档案与地图线索文案
- 优化 GitHub 展示文档与作品集复盘材料
- 如果后续目标发生变化，再单独评估是否进入高保真原型阶段

当前不会接入真实登录、预约、数据库、CMS、后台或 API，也不会把项目包装成真实已上线游戏。
