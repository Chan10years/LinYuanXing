# C1 Submission Checklist

本检查清单用于《临渊行》C1 参赛结构补齐阶段的自检。项目定位保持为纯静态国风志怪叙事游戏官网作品集，不接入真实登录、真实预约、数据库、后台、CMS 或 API。

## Page Structure

- [ ] 首页 `index.html` 保持可访问，Hero、PV、角色、地图、Lore 核心布局未被破坏。
- [ ] 栏目页结构清晰，后续新增页面应优先从 `characters.html` 开始。
- [ ] 详情页从所属栏目进入，不孤立存在。
- [ ] 三层结构符合“首页 / 栏目页 / 详情页”的浏览逻辑。
- [ ] 每个页面都有明确标题、主题说明和返回路径。
- [ ] 新增页面数量克制，不为凑数量制造空页面。

## Link Flow

- [ ] 首页可以进入主要栏目页。
- [ ] 栏目页可以返回首页。
- [ ] 栏目页可以进入对应详情页。
- [ ] 详情页可以返回所属栏目页。
- [ ] 详情页可以返回首页或全站主要导航。
- [ ] `404.html` 保持可访问，并提供回到首页的路径。
- [ ] 所有内部链接使用相对路径。
- [ ] 页面跳转不依赖真实路由服务、后端重写或 JavaScript 强绑定。

## Static Boundary

- [ ] 不新增真实登录功能。
- [ ] 不新增真实预约功能。
- [ ] 不新增数据库。
- [ ] 不新增后台管理。
- [ ] 不新增 CMS。
- [ ] 不新增 API 请求。
- [ ] 不收集用户信息。
- [ ] 不提供真实支付、下载、短信、邮件、上传或账号系统。
- [ ] `login.html` 与 `reserve.html` 保持静态占位说明属性。

## External Dependency Check

- [ ] 页面不依赖外链图片。
- [ ] 页面不依赖外链视频作为关键展示资源。
- [ ] 页面不依赖外链字体才能完成基本展示。
- [ ] 页面不依赖第三方脚本完成核心浏览。
- [ ] 参赛展示所需主要资源都位于仓库内。
- [ ] Netlify / GitHub Pages 等静态托管环境下不需要额外构建命令。

## Asset Path Check

- [ ] 图片、视频、图标等资源路径使用相对路径。
- [ ] 文件名大小写与引用完全一致。
- [ ] 新增资源放入 `assets/` 或约定的资源目录，不散落在根目录。
- [ ] 新增页面引用资源前确认文件实际存在。
- [ ] 不移动现有核心资源，避免破坏首页展示。
- [ ] 大体积资源后续可在 `docs/asset-notes.md` 中记录优化建议。

## Mobile And Responsive

- [ ] 首页在移动端保持可读。
- [ ] 新增栏目页在移动端不横向溢出。
- [ ] 新增详情页正文、卡片、图片在小屏幕下可正常浏览。
- [ ] 导航入口在移动端可点击。
- [ ] 弹层或按钮不会遮挡主要内容。
- [ ] 字号、行距和按钮点击区域适合移动端展示。

## Documentation

- [ ] `README.md` 记录 Stage 4B 当前进入 C1 参赛结构补齐阶段。
- [ ] `docs/stage-log.md` 记录 Stage 4B Day 1。
- [ ] `docs/c1-information-architecture.md` 描述三层信息架构。
- [ ] `docs/c1-submission-checklist.md` 可用于后续自检。
- [ ] 项目边界在文档中保持一致：静态概念官网，不是真实运营游戏服务。
- [ ] 后续新增页面时同步更新相关文档。

## Competition Materials

- [ ] README 展示信息完整，能说明项目定位、技术栈、边界和展示地址。
- [ ] 展板内容能概括项目主题、页面结构、视觉特色和实现方式。
- [ ] 海报内容与官网视觉风格保持一致，不夸大真实业务能力。
- [ ] 宣讲视频能展示首页、栏目页、详情页的完整浏览路径。
- [ ] 视频讲解明确说明项目为静态概念官网作品集。
- [ ] 参赛材料中的链接可打开，且不依赖本地绝对路径。
- [ ] 截图、展板、海报、视频中的页面状态与仓库当前版本一致。
